## Overview

GAT follows a three-phase sequential pipeline architecture:

```
Requirements Document (RD)
        │
        ▼
┌─────────────────┐
│  Requirements    │  → review.md
│  Phase           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Hiring          │  → crew.yaml
│  Phase           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Execution       │  → project output
│  Phase           │
└─────────────────┘
```

Each phase is driven by CrewAI agents whose configuration comes from YAML files.

## Module Descriptions

### `config_loader.py`
- `load_models(path)` – Parses `models.yaml` into a dict.
- `load_crew(path)` – Parses a crew YAML file.
- `validate_crew(data, models)` – Validates agent/task structure and model references.

### `work_log.py`
- `append_run(...)` – Appends a structured Markdown entry to `{log_dir}/{phase}/{agent_name}.md`.

### `phases/requirements_phase.py`
Creates a Senior Consultant + Requirements Engineer crew. Outputs a review document with feasibility assessment, clarification questions, and suggested improvements.

### `phases/hiring_phase.py`
Uses a Hiring Manager agent to generate a crew YAML from requirements. Includes retry logic and markdown fence stripping. Always includes a Technical Author and an integration-test task.

### `phases/execution_phase.py`
- Creates a shared Python venv.
- Ports `ShellTool` and `PythonREPLTool` (from `agentic_scanner`) as `crewai.tools.BaseTool` subclasses.
- Builds agents and tasks from crew YAML, runs via `Crew.kickoff()`.

### `cli.py`
Three subcommands: `requirements`, `hire`, `run`. Loads models, dispatches to the appropriate phase, prints result.

## Data Flow

```
User → CLI (cli.py)
        │
        ├─ requirements → requirements_phase.run()
        │                   ├─ reads RD
        │                   ├─ creates CrewAI agents (LLM from models.yaml)
        │                   ├─ crew.kickoff()
        │                   ├─ writes review.md
        │                   └─ writes work logs
        │
        ├─ hire → hiring_phase.run()
        │           ├─ reads RD
        │           ├─ prompts LLM to generate crew YAML
        │           ├─ validates with config_loader.validate_crew()
        │           ├─ writes crew.yaml
        │           └─ writes work logs
        │
        └─ run → execution_phase.run()
                    ├─ loads crew.yaml via config_loader
                    ├─ creates shared venv
                    ├─ instantiates ShellTool / PythonREPLTool
                    ├─ builds Agent + Task objects from YAML
                    ├─ crew.kickoff()
                    └─ writes work logs
```

## Configuration

### `models.yaml`

Maps logical aliases to Ollama model strings:

```yaml
models:
  large:  "ollama/qwen3.5:35b-ctx32k"
  dev:    "ollama/qwen3.5:9b-ctx16k"
  tester: "ollama/mistral:ctx16k"
```

### Crew YAML (generated by hiring phase)

```yaml
agents:
  - name: Developer
    role: Developer
    goal: Write code
    backstory: Expert coder
    model: dev
    tools: [shell, python_repl]
  - name: Tester
    role: Tester
    goal: Test the project
    backstory: QA specialist
    model: tester
    tools: [shell]

tasks:
  - name: implement
    description: Implement the feature
    expected_output: Working code
    agent: Developer
  - name: integration_test
    description: Run integration tests
    expected_output: Test report
    agent: Tester
```
